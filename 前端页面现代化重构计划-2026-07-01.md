## 一、执行原则

- 颜色系统统一切到 Kumo 风格语义 token：`--kumo-bg-base`、`--kumo-text-default`、`--kumo-bg-brand`、`--kumo-bg-elevated`、`--kumo-hairline`、状态 tint 等。
- 支持 `<html data-mode="light|dark">`，同时兼容系统暗色偏好。
- 参考 React Bits 的高级但克制的交互：磁吸/抬升卡片、滚动揭示、流体网格背景、文字渐入、列表错峰入场。
- 现有项目仍是 Vue 3 + Vite + JavaScript + Vuex + Element Plus。本轮先在不破坏业务链路的前提下重构展示层，并补 Tailwind v4 语义 token 配置作为后续接入基础。
- 页面重构优先保留真实接口，不引入 mock、假成功或静默 fallback。

## 四、集成注意事项

- 当前项目尚未安装 Tailwind CSS，本轮新增 `tailwind.config.js` 是按需求补齐的语义色配置建议与后续接入点；现有页面实际运行仍通过 CSS variables 和普通 scoped CSS。
- 当前项目仍使用 JavaScript + Vuex；未强行迁移 TypeScript/Pinia，避免扩大业务风险。后续如迁移，应先补类型边界和 store 兼容层。
- 新增主题切换会写入 `localStorage.themeMode` 并设置 `<html data-mode="light|dark">`；后台仪表盘同步写入 `adminTheme` 与同一 `data-mode`，管理端/用户端共享 Kumo 主题变量。
- 本轮未新增 GSAP/@vueuse/motion 依赖，React Bits 风格交互用 CSS transform、transition、keyframes 实现，减少包体和运行时成本。
