# 交付阻断问题与修复记录 - 2026-06-13

## 本轮用户反馈

1. 更换头像失败，报错路径落到 Tomcat 临时目录。
2. 文章 Markdown 上传图片会退出登录。
3. 很多接口报错后会自动退出登录。
4. 登录状态过期太快，评论时偶发退出登录。
5. 发布文章填写摘要后发布不了。
6. 评论可以无限点赞。
7. 关注作者后仍显示“关注作者”。
8. 管理员通知是摆设，无法查看。
9. 私信功能没有实现。

## 根因与修复

| 问题 | 根因 | 修复 |
| --- | --- | --- |
| 头像上传写入 Tomcat 临时目录 | 头像接口使用相对路径 `Paths.get(uploadPath, "avatars")`，运行时会按容器工作目录解析 | 统一使用上传根目录绝对路径，头像保存到 `uploads/images/avatars/`，访问路径为 `/uploads/images/avatars/...` |
| 上传图片、评论等操作容易退出登录 | JWT 配置值按秒填写，但代码按毫秒使用，默认 token 实际约 86 秒过期；前端 401 会清空所有登录态并强制跳转 | JWT 过期单位做秒/毫秒兼容，本地默认改为 7 天；前端 401 改为按用户/管理员请求分别清理，公开页不强制跳转 |
| 通知未读数/管理员通知不可用 | 通知拦截器只拦截 `/api/notifications/*`，未覆盖 `/api/notifications/unread/count`；管理员顶部铃铛写死数字 | 改为拦截 `/api/notifications/**`；管理员顶部通知读取真实未读数和最近通知，点击进入消息中心 |
| 评论无限点赞 | 前端每次点赞后只做本地 `likes++`，没有读取后端真实点赞状态 | 评论加载后同步 `/likes/COMMENT/{id}/info`，点击时按当前状态点赞/取消点赞，并用后端返回计数刷新 |
| 关注作者按钮不变 | 文章详情按钮文字写死为“关注作者”，没有加载作者关注状态 | 加载文章后读取 `/follows/USER/{authorId}/info`，按钮显示“已关注/关注作者/作者本人” |
| 文章摘要发布异常 | 编辑器界面使用 `article.category`，提交层使用 `categoryId`，字段容易漂移；自动摘要逻辑也不稳定 | 编辑器统一维护并提交 `categoryId`，摘要为空时从正文生成 200 字以内摘要，手写摘要按 500 字限制校验 |
| 私信没有实现 | 没有独立私信表和接口，消息中心只有通知展示 | 新增站内私信发送接口 `/api/notifications/messages`，以 `DIRECT_MESSAGE` 通知形式投递；消息中心支持写私信和回复私信 |
